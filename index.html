Automate SSL Certificates
with DNS Verification

Self-hosted DNS challenge server for ACME. Get wildcard certificates for any domain with a simple CNAME record.

Get Started View on GitHub
# Issue a wildcard certificate in one command $ HTTPREQ_ENDPOINT=https://api.dnsall.com \ HTTPREQ_USERNAME=myuser \ HTTPREQ_PASSWORD=$API_KEY \ lego --dns httpreq --domains "*.example.com" --accept-tos run [INFO] [*.example.com] acme: Obtaining certificate [INFO] [*.example.com] acme: use dns-01 solver [INFO] [*.example.com] The server validated our request [INFO] [*.example.com] Server responded with a certificate.

Built for automation

Everything you need to manage DNS challenges at scale

☍

CNAME Delegation

Point _acme-challenge to httpdns via CNAME. No need to grant CA clients access to your production DNS.

♟

Multi-User Isolation

Each user gets a unique nanoid subdomain per domain. Multiple users can independently manage the same domain.

⚿

API Key Authentication

Separate API keys for CLI tools. Web dashboard uses password login. Regenerate keys anytime from the UI.

☷

Web Dashboard

Add domains, copy CNAME records, view active TXT records, and get pre-filled environment variables.

⧉

Single Binary

Go binary with embedded React SPA. Deploy one file — no Node.js, no nginx, no separate frontend.

⚙

Admin API

Manage all users and domains via API key. Create users, assign domains, and monitor records programmatically.

How it works

Four steps from zero to wildcard certificate

Register

Create an account and add your domain in the web dashboard

Set CNAME

Copy the CNAME record and add it to your domain's DNS

Configure

Set HTTPREQ_ENDPOINT and credentials in your environment

Issue

Run lego or configure Traefik — certificates auto-renew

Get started in minutes

Deploy with Docker or build from source

Docker Compose

# docker-compose.yml services: httpdns: image: zzci/httpdns:latest restart: unless-stopped ports: - "53:53/udp" - "53:53" - "3000:3000" volumes: - ./data:/app/data

Traefik Integration

# traefik.yml certificatesResolvers: letsencrypt: acme: email: admin@example.com storage: /data/ssl/acme.json dnsChallenge: provider: httpreq propagation: disableChecks: true # environment variables LEGO_DISABLE_CNAME_SUPPORT=true HTTPREQ_ENDPOINT=https://api.dnsall.com HTTPREQ_USERNAME=myuser HTTPREQ_PASSWORD=<api_key>
