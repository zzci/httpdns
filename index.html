httpdns

DNS TXT record management server for ACME DNS-01 challenges. Works with lego, Traefik, and any httpreq-compatible client.

GitHub Docker Hub

Features

CNAME Delegation

Delegate _acme-challenge records via CNAME. No need to give CA clients access to your primary DNS zone.

Multi-User

Each user gets isolated domains with unique nanoid subdomains. Different users can manage the same domain independently.

API Key Auth

Separate API keys for httpreq Basic Auth. Web login uses password, CLI uses API key.

Web Dashboard

Manage domains, view CNAME records, check active TXT records, and copy httpreq configuration.

Single Binary

SPA frontend embedded in the Go binary. Deploy one file, no separate frontend server needed.

Multi-Arch Docker

Pre-built images for linux/amd64 and linux/arm64. Works on cloud VMs and Raspberry Pi.

How It Works

Quick Start

# docker-compose.yml services: httpdns: image: zzci/httpdns:latest ports: - "53:53/udp" - "53:53" - "3000:3000" volumes: - ./data:/app/data

Usage with lego

LEGO_DISABLE_CNAME_SUPPORT=true \ HTTPREQ_ENDPOINT=https://api.dnsall.com \ HTTPREQ_USERNAME=myuser \ HTTPREQ_PASSWORD=<api_key> \ lego --dns httpreq --dns.propagation-disable-ans \ --domains example.com --domains "*.example.com" \ --email admin@example.com --accept-tos run
